# FTP 密码特殊字符处理指南

## 问题描述
当FTP密码包含特殊字符（如 `%`, `"`, `(`, `)`, `$`, `*`, `?`, `[`, `]`, `{`, `}`, 等）时，在使用 `lftp` 命令行工具进行部署时可能会出现解析错误。

## 解决方案
当前的部署脚本 (`scripts/deploy-frontend-to-aliyun-ftp.js`) 已经实现了对特殊字符的安全处理：

1. **字符串转义**：对用户名和密码中的特殊字符进行转义处理
2. **临时脚本文件**：使用临时脚本文件而非命令行参数传递密码，避免在进程列表中暴露密码
3. **安全清理**：执行完毕后自动清理临时脚本文件

## 特殊字符处理规则
脚本会自动处理以下特殊字符：
- 反斜杠 `\` → `\\`
- 双引号 `"` → `\"`
- 美元符号 `$` → `\$`
- 反引号 `` ` `` → `` \` ``

## 配置示例
在 `.env` 文件中配置包含特殊字符的密码：

```env
FTP_HOST=your-ftp-server.com
FTP_USER=your-username
FTP_PASS="%)(AabhMXUsyiA5"
FTP_PORT=21
FTP_REMOTE=/
```

或者使用环境变量：

```bash
export FTP_HOST='your-ftp-server.com'
export FTP_USER='your-username'
export FTP_PASS='%)(AabhMXUsyiA5'
npm run deploy:aliyun:ftp
```

## 注意事项
1. 即使密码包含特殊字符，也无需手动转义，脚本会自动处理
2. 请确保已安装 `lftp` 工具
3. 部署过程中，密码不会在命令行或日志中明文显示
4. 临时脚本文件会在部署完成后自动删除

## 验证部署
部署完成后，可通过以下步骤验证：
1. 检查文件是否正确上传到FTP服务器
2. 访问网站确认功能正常
3. 查看FTP服务器上的文件权限是否正确
